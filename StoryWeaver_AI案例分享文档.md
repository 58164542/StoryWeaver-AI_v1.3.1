# StoryWeaver AI：AI驱动的智能视频内容生成平台

## 一、问题定义（30秒）

### 1.1 遇到的实际问题

**问题描述：**

传统视频内容生产流程存在严重的效率痛点：
- 将小说、文案、剧本转化为视频作品需要**人工分镜、多次修改、反复生成**
- 角色建模、场景设计、视频合成等环节需要**高度专业化技能**
- 修改一个细节往往需要重新拆解、重新生成整套资产，**重复劳动严重**

**问题影响：**
- **痛点：**
  - 创意到成片的周期长，难以快速迭代
  - 创作者需要学习多个专业工具，学习成本高
  - 质量不稳定，依赖个人经验和技能差异大
  - 后期修改成本极高，难以满足快速变更需求

### 1.2 为什么选择用AI解决

**AI的核心价值：**
1. **理解语义**：通过大模型理解文本内容，自动识别关键角色、场景、元素
2. **智能拆解**：将复杂的创意自动转化为可执行的分镜和视频参数
3. **并行生成**：多个AI能力同时工作（资产生图、分镜拆解、视频生成）
4. **持续优化**：通过样本积累不断改进提示词和模型选择
5. **可复用性强**：一套流程适配多种项目类型（真人、2D、3D等）

这些问题传统方法无法解决：
- ❌ 手工拆解：速度慢、容易遗漏细节
- ❌ 工具堆砌：需要学习6+个专业软件
- ❌ 模板库：灵活性差，难以定制
- ✅ **AI方案**：一键生成、智能优化、反复迭代

## 二、解决方案（2分钟）

### 2.1 工具选择

**使用的AI工具：**
- ☑️ Claude Code（架构设计、代码实现）
- ☑️ Claude Haiku/Sonnet（多模型组合）
- ☑️ 豆包2.0（文本理解、资产分析）
- ☑️ Banana2.0（图像生成）

**为什么选择这个工具组合：**

| 工具 | 用途 | 选择原因 |
|-----|-----|--------|
| Claude | 架构设计 + 工程化 | 擅长复杂系统设计、代码质量、工程最佳实践 |
| 豆包2.0 | 文本分析 | 长文本理解能力强，适合小说/剧本拆解 |
| Banana2.0 | 图像生成 | API 稳定，支持参考图，保证角色一致性 |

### 2.2 实现过程

**核心步骤：**

1. **需求分析与架构设计**
   - 定义5种项目类型（真人、2D解说、3D解说、精品2D、精品3D）
   - 设计7阶段工作流（资产提取→资产生图→分镜拆解→分镜生图→视频生成→配音→导出）
   - 规划多主机架构支持本地和云端部署

2. **核心工作流实现**
   - **第1-2阶段**：用豆包2.0分析小说，自动提取角色、场景、道具资产，调用图像生成API生成参考图
   - **第3-4阶段**：将提取的资产信息和剧本内容送入豆包2.0/火山引擎，生成分镜拆解，再生成分镜图像
   - **第5-7阶段**：调用火山引擎 API 生成视频，集成第三方配音服务，最后导出完整视频

3. **迭代优化与新功能**
   - **2月27日**：新增角色变体功能（同一角色不同服装状态）
   - **2月28日**：分镜双提示词改造（imagePrompt 用于生图，videoPrompt 用于生视频）
   - **实时自动保存**：500ms防抖自动保存到后端，支持多客户端协作

**遇到的主要卡点及解决方法：**

| 卡点 | 问题 | 解决方案 |
|-----|-----|--------|
| **AI理解一致性** | 不同AI模型对同一文本理解不同 | 建立统一的提示词框架，在DEFAULT_GLOBAL_SETTINGS中硬编码5类项目的提示词模板，确保输出格式一致 |
| **实时协作** | 多用户同时编辑时状态不一致 | 采用500ms防抖 + 后端LowDB存储，确保每次修改都自动保存并同步 |

### 2.3 关键技巧/启发

**架构设计层面：**
- **分离关注点**：分镜拆解和图像生成是独立的任务队列，可并行执行，提升整体效率
- **提示词工程**：在应用层硬编码多套提示词模板，而不是依赖前端输入，确保输出质量可控
- **渐进式增强**：先实现基础流程，再逐步添加变体、双提示词等高级功能，降低初期复杂度

**工程实践层面：**
- **自动保存机制**：防抖+后端存储，让用户无感知地享受自动备份和多设备同步
- **数据兼容性策略**：新功能上线时必须考虑旧数据迁移，在加载时而不是保存时进行转换
- **日志驱动调试**：对关键路径（参考图传输、AI响应解析）添加详细日志，大幅降低调试成本

**与AI协作层面：**
- **多模型互补**：Gemini 长文本强，火山引擎 API 文档详细，选择各自擅长的领域
- **结构化输出**：定义严格的 Schema（JSON schema validation），确保AI输出可被程序解析
- **错误恢复**：为每个AI调用添加重试逻辑和备用方案，提升系统稳定性

## 三、结果验证（1.5分钟）

### 3.1 实际效果

**Demo展示要点：**
- ✅ 用户输入小说文本 → 自动提取角色、场景、道具（5秒内）
- ✅ 点击"生成分镜" → AI自动拆解场景，生成详细的分镜信息和图像（1-2分钟）
- ✅ 支持实时编辑分镜、修改提示词、重新生成特定帧
- ✅ 支持多种项目类型快速切换，同一内容可生成5种风格的视频
- ✅ 后端自动保存，支持多设备无缝同步

**量化效果：**
  - **创新速度加快**：可快速验证创意，从想法到初稿仅需1小时
  - **可复用性**：一套系统适配5种项目类型，扩展成本低

### 3.2 业务价值

**直接价值：**
1. **成本大幅下降**：从需要3-5人的专业团队 → 1人加AI可以完成同样的工作
2. **产能显著提升**
3. **创作民主化**：非专业创作者也能产出专业级视频内容，扩大用户基数
4. **快速迭代**：修改一个细节不需要重来，可在5分钟内完成，支持A/B测试

**可复用性：**
- **团队推广价值**：整套工作流可直接部署到其他内容创作团队
- **跨项目复用**：架构设计适配5种项目类型，新增项目类型只需更新提示词
- **技能沉淀**：提示词工程、多模型协调、自动化工作流等经验可成为团队资产
- **生态扩展**：支持接入新的AI模型、图像生成服务、视频特效库等

**长期价值：**
- 可作为SaaS产品方向，面向内容创作者/营销团队收费
- 技术可输出到相邻领域（动画、教育、短视频平台）
- 积累的提示词库和模型评估体系具有重要的商业价值

## 四、下一步计划（1分钟）

### 4.1 当前局限

**技术层面：**
- **视频生成**：seedance api未开放，需要后续接入
- **模型稳定性**：某些复杂场景的AI理解可能不准确，需要人工微调
- **成本控制**：调用多个付费API，大规模使用时成本较高

**功能层面：**
- **音效和背景音乐**：目前只支持配音，缺少自动化的音效和音乐选择
- **字幕优化**：字幕位置、大小、样式目前还需手工调整

### 4.2 优化方向

**近期：**
1. **引入音乐和音效库**：集成Epidemic Sound或本地音库，实现自动选择和时间对齐
2. **提升视频生成速度**：探索多模型并行调用，或切换到更快的视频生成服务
3. **优化提示词机制**：提升提示词改编效率

**中期（3-6月）：**
1. **多语言支持**：实现自动的多语言字幕和配音（通过翻译API + TTS）

**长期（6-12月）：**
1. **用户反馈闭环**：积累视频质量数据，持续优化提示词和模型选择
